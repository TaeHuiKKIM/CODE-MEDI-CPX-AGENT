# CODE MEDI CPX Agent - 해커톤 우승 전략 가이드

이 문서는 무박 2일 해커톤에서 팀원들이 효율적으로 협업하고, 심사위원들에게 강렬한 인상을 남길 수 있는 필승 전략을 담고 있습니다.

## 1. 기획 아이디어: "채점자 겸용 하이브리드 CPX 에이전트"

평범한 '의학 챗봇'을 넘어, 실제 까다로운 환자처럼 행동하고 진료 후에는 의대생을 평가해주는 교육용 플랫폼으로 접근합니다.

- **환자 모드 (진료 중)**: LLM이 설정된 시나리오에 따라 환자 연기를 합니다.
- **평가자 모드 (진료 종료 후)**: 방금 전 대화 기록(Log)과 의대생들이 제공한 '채점 기준표'를 비교하여 피드백을 제공합니다. ("의사가 과거력을 묻지 않았음", "공감하는 표현이 부족했음" 등)

## 2. 세부 개발 전략 (에이전트 설계법)

### ① '묻지 않은 건 절대 먼저 말하지 않는' 룰 설정
LLM의 기본 성향인 '친절함'을 억제해야 합니다. "배가 아파요"라고 했을 때 스스로 원인까지 술술 불어버리지 않도록, `main.py`의 프롬프트에 **"의사가 명시적으로 묻지 않은 정보는 절대 먼저 말하지 마라"**는 엄격한 규칙이 이미 구현되어 있습니다.

### ② 시나리오 퀵 주입 구조 (Vibe Coding)
대회 당일 '상세 세부 주제(특정 질병 시나리오)'가 공개됩니다. 
의대생 팀원들은 **"환자 프로필, 주증상, 숨겨진 병력, 성격, 체크리스트"**를 정리해 주시면 됩니다. 
개발자 팀원은 `main.py` 최상단의 `PATIENT_SCENARIO` 변수만 수정하면 즉각적으로 새로운 에이전트가 탄생합니다!

### ③ 페르소나/감정 상태의 부여 (Rapport 시스템)
실제 CPX 시험에서 가장 중요한 것은 **'공감 능력(Rapport)'**입니다. 
본 시스템은 의사가 "많이 아프셨겠네요" 같은 공감 표현을 하면 환자의 호감도 수치가 올라가고 대답을 더 협조적으로 하도록 동적 감정 상태(State) 변수를 구현했습니다. 프론트엔드의 화려한 Progress Bar로 이를 시각화하여 심사위원에게 어필합니다.

## 3. 개발자를 위한 CPX (진료수행평가) 도메인 지식 가이드

의대생 팀원들이 주로 사용할 의학적 뼈대를 개발자도 알아야 합니다.

- **1단계 C.C (Chief Complaint, 주증상)**: 환자가 방문한 이유. AI의 초기 상태 설정값.
- **2단계 PI (Present Illness, 현병력) - ★가장 중요**:
  의대생들은 **'OLDCAARTS(올드카츠)'**라는 알고리즘으로 질문합니다.
  (Onset 언제, Location 어디가, Duration 얼마나 오래, Character 양상, Aggravating/Alleviating 악화/완화 조건, Radiation 방사통, Timing 시간, Severity 통증 척도)
  👉 **개발자 포인트**: 이 8가지 항목이 바로 **AI가 숨기고 있다가 의사가 정확히 물어봐야만 대답할 핵심 변수(Variables)**입니다.
- **3단계 History (과거력/가족력/사회력)**: 기저질환, 직업, 술/담배 등. AI의 백그라운드 설정값.
- **4단계 ROS (Review of Systems, 전신 상태 파악)**: 주증상 외 발열/체중감소 등 다른 시스템 점검.

## 4. 해커톤 필승을 위한 팀워크 & 타임라인 팁

의대생 3명 + 컴공생 3명의 완벽한 역할 분담 체제를 유지하세요.

- **의대생 (도메인 전문가)**:
  1. 대회 당일 주제가 나오면 즉시 현실적인 '가상의 환자 케이스(시나리오)' 구축.
  2. CPX 채점 기준표(루브릭) 작성.
  3. LLM의 연기를 계속 테스트하며 "이건 너무 인위적이다", "이 부분은 훌륭하다"며 프롬프트 엔지니어링 QA 진행.
- **컴퓨터학부 (엔지니어)**:
  1. `main.py` 백엔드 구축 및 프롬프트 파라미터화.
  2. 프론트엔드 퍼블리싱 (음성 인식 STT/TTS 적용 및 화려한 애니메이션/차트 렌더링).
  3. Cloudtype 배포 관리 및 시연 환경 세팅.

## 5. Cloudtype.io 1분 배포 가이드

해커톤 심사 전, 심사위원들이 직접 스마트폰이나 노트북으로 접속해 볼 수 있게 배포합니다.
1. 이 프로젝트(`cpx-agent`)를 본인의 GitHub 레포지토리에 Push합니다.
2. Cloudtype.io 에 접속 및 로그인.
3. [새 프로젝트] -> [GitHub 저장소 가져오기] 클릭.
4. 방금 올린 레포지토리 선택 후, 프레임워크를 **FastAPI**로 선택.
5. 포트 번호를 **8000**으로 설정.
6. [환경변수] 탭에 `GEMINI_API_KEY`와 그 값을 입력.
7. [배포하기] 버튼 클릭! 1분 뒤 생성되는 도메인 URL을 팀원 및 심사위원에게 공유하세요.
