# 🩺 CODE MEDI CPX Agent

**의대생의 임상수행능력평가(CPX)를 완벽하게 훈련하고 평가하는 LLM 기반 가상 환자 플랫폼**

본 프로젝트는 단순한 텍스트 챗봇을 넘어, 의대생이 실제 환자를 마주했을 때 필요한 **문진(History Taking), 신체진찰(Physical Examination), 위험신호(Red Flag) 감지, 그리고 환자와의 라포(Rapport) 형성**까지 종합적으로 훈련하고 채점해 주는 혁신적인 하이브리드 에이전트입니다.

## 🌟 1. 프로젝트 개요 (CPX 전반)

임상수행능력평가(CPX)에서 좋은 점수를 받기 위해서는 단순히 질병을 맞추는 것이 아니라, 환자의 숨겨진 사연을 이끌어내고 응급 상황을 빠르게 판단하는 능력이 필수적입니다. 
본 에이전트는 다음과 같은 철학으로 설계되었습니다.

1. **채점자 겸용 하이브리드 에이전트**: 진료 중에는 완벽한 페르소나를 가진 까다로운 환자로 연기하며, 진료가 종료되면 대화 로그를 분석하여 CPX 채점 기준(루브릭)에 맞춰 냉철한 피드백을 제공합니다.
2. **동적 감정 시스템 (Rapport & Stress)**: 의사가 공감 없이 기계적인 질문만 던질 경우 환자의 스트레스가 상승하여 중요한 정보를 숨깁니다. 공감해 줄 때만 핵심 단서를 제공합니다.
3. **묻지 않은 건 말하지 않는다**: 실제 환자처럼 의사가 OLDCAARTS(증상의 양상, 악화 요인 등)를 구체적으로 묻지 않으면 절대 원인을 먼저 발설하지 않습니다.

## 🔥 2. 주요 핵심 기술 (Hackathon Features)

### ① 신체진찰 (Physical Examination) 지문 시스템
가상 챗봇의 한계(직접 만져볼 수 없음)를 혁신적으로 해결했습니다. 
의사가 문진 도중 **"청진기로 폐 소리를 들어보겠습니다"** 또는 **"배를 눌러보겠습니다"**라고 말하면, AI가 환자로서의 반응(비명 등)을 함과 동시에 괄호 `[ ]` 안에 **객관적인 신체진찰 소견(Physical Exam Findings)**을 시스템 메시지로 출력해 줍니다.
> *예: "아얏! 거길 누르니까 너무 아파요!" `[신체진찰 결과: 우하복부 압통(RLQ tenderness) 양성, 반발통 양성]`*

### ② 위험신호 (Red Flag) 실시간 인터럽트 시스템
환자가 가슴 답답함(심장혈관 폐색 의심 등)과 같은 치명적인 **Red Flag**를 호소함에도 불구하고, 의사가 쓸데없는 추가 문진만 늘어놓으며 시간을 낭비하면 환자의 상태가 급격히 악화됩니다. 
> *"선생님... 언제까지 질문만 하실 건가요? 저 진짜 숨이 넘어가고 가슴이 찢어질 것 같아요!!"* 와 같이 대화를 강제로 끊고 응급 처치(ECG, 응급실 이송 등)를 유도하여 의사의 응급 상황 대처 능력을 테스트합니다.

## 🌡️ 3. 이번 대회 출제 모듈: "발열 (Fever)"

발열은 전신 질환의 신호이므로, 감기부터 중증 감염병, 종양까지 다양한 원인이 존재합니다. 이번 프로젝트의 메인 데모 시나리오는 **발열 모듈**에 집중합니다.

* **환자의 숨겨진 특성**: 열이 펄펄 나지만, 정작 핵심이 되는 **부끄러운 증상(예: 배뇨통)**이나 **특별한 과거력(예: 최근 동남아/아프리카 여행력)**은 의사가 직접 묻기 전까지 철저히 숨깁니다.
* **발열 모듈에서의 Red Flag 감지**: 발열과 함께 의식 저하, 심한 목 뻣뻣함(수막염 의심)이나 호흡 곤란(중증 폐렴/패혈증)이 동반될 경우, 문진을 즉각 멈추고 신체진찰 및 응급 처치로 넘어가도록 몰아가는 시나리오를 구성했습니다.
* **신체진찰 연계**: 의사가 목 림프절 촉진, 흉부 청진, 복부 타진 등을 지시할 때 발열 원인을 좁힐 수 있는 결정적 단서(Sign)를 제공하도록 프롬프트가 세팅되어 있습니다.

## 💻 실행 방법 (Local Setup)

1. 패키지 설치: `pip install -r requirements.txt`
2. 환경변수 세팅: `.env` 파일에 `GEMINI_API_KEY=당신의키` 입력
3. 서버 실행: `uvicorn main:app --reload --host 0.0.0.0 --port 8000`
4. 브라우저에서 `http://localhost:8000/static/index.html` 접속
